# 网文去AI味检测流程

## Gate 1: 词汇密度检测

检查以下AI高频词在每千字中出现的次数：
- "宛如/犹如/仿佛" > 2次/千字 → FAIL
- "不禁/不由自主" > 1次/千字 → FAIL
- "微微/缓缓/徐徐/悄然" > 3次/千字 → FAIL
- "竟然/居然" > 2次/千字 → FAIL
- "意味深长/若有所思" > 1次/千字 → FAIL

## Gate 2: 句式单一检测

- 连续3句以上使用"主语+说道"结构 → FAIL
- 连续3段以人名开头 → FAIL
- 段落平均长度偏差 < 20%（过于均匀） → FAIL

## Gate 3: 情感表达检测

- 使用"眼中闪过""瞳孔微缩""嘴角上扬"等模板化表情 > 3次/千字 → FAIL
- 心理活动全用"心中暗道""心想"引导 → FAIL
- 缺少具体的、个性化的感官描写 → FAIL

## Gate 4: 对话自然度检测

- 对话中频繁使用书面语 → FAIL
- 所有角色说话风格雷同 → FAIL
- 对话缺少语气词、省略、中断等自然特征 → FAIL

## Gate 5: 节奏检测

- 全文节奏均匀，无快慢变化 → FAIL
- 动作场景缺少短句和碎句 → FAIL
- 缺少段落间的留白和呼吸感 → FAIL

## Gate 6: 重复模式检测

- 相邻段落使用相同句式结构 → FAIL
- 同一场景中重复使用相同描写套路 → FAIL
- 过渡段使用"与此同时""正因如此"等AI连接词 → FAIL

## Gate 7: 情绪冗余检测（新增）

核心原则：**AI味的真病根是描写内容通用化，最常见表现是情绪冗余。去掉的是"不提供新信息的废话"，不是去掉比喻/修辞。**

### 检测方法

以本章为单位，**逐句判断**当前句子是否为本场景已积累信息提供了新的具体信息。

### 检测规则

| 编号 | 规则 | 例子（PASS / FAIL） |
|---|---|---|
| 7.1 | 同一情绪状态被多次显式声明，后文无新信息 | "他感到愤怒。他紧紧攥着拳头，胸膛起伏着。" → 后句FAIL（愤怒已暗示，攥拳是愤怒的通用模板化表现） |
| 7.2 | 抽象情绪词包裹具体动作，拆开写后冗余 | "他心中涌起一阵悲伤的浪潮，泪水模糊了视线。" → 前句FAIL（"悲伤的浪潮"是AI式情绪修辞，"泪水模糊了视线"已足够传递情绪） |
| 7.3 | 场景描写完成后，追加一句情绪总结 | "桌上只有一碗冷粥。他感到一阵凄凉。" → 后句FAIL（冷粥本身已传递凄凉，情绪总结是多余的） |
| 7.4 | 已经通过动作/对话/环境间接表达的情绪，又用心理活动再写一遍 | "他握着那张泛黄的照片，指节发白。他心想：这些年真不容易啊。" → 后句FAIL（指节发白已间接表达，心理活动是重复） |
| 7.5 | 通用化情绪比喻，无场景特异性 | "他的心情像打翻了五味瓶" → FAIL（任何场景都能用，不是基于本章具体情节推导出的） |
| 7.6 | 角色已明确表态后，追加心理活动解释 | "'我不同意。'他在心里想：这方案根本行不通。" → 后句FAIL（对话已表态，心理活动是冗余解释） |
| 7.7 | 跨段重复同一情绪点 | "他意识到自己被骗了，怒火中烧。" …5段后… "他想起来龙去脉，更加愤怒了。" → 后句FAIL（愤怒状态已在前文确立，无新触发事件时再次声明是冗余） |

### 判断原则

1. **范围**：读本章全文判断，不是只看前后两句
2. **标准**：这句话是否让读者知道了**之前不知道的事情**或**之前不知道的具体细节**
3. **豁免**：
   - 通过重复来强化节奏感的刻意重复（如排比） → PASS
   - 相似情绪因**新触发事件**而再次激发 → PASS（需标注触发事件）
   - 关键信息的递进式展开（先说结果，倒叙原因） → PASS
4. **不豁免**：
   - 发现句子没有提供新信息，只是换句话说了前文已经表达过的情绪/状态 → FAIL

## 真人写作基准

真人作者的文本特征：
- 词汇多样，很少重复使用同一修饰词
- 句式长短交错，节奏有变化
- 对话有口语感，不同角色语气不同
- 感官描写具体且有个人风格
- 段落长度不均匀，有呼吸感
- 心理活动融入动作和环境，不单独成段
