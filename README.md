# Show Me The Story — AI 小说生成器（白泽改造版）

> 基于 [Nigh/show-me-the-story](https://github.com/Nigh/show-me-the-story) 深度改造。
> 以下是与原版的主要差异，未列出的功能与原版一致。
> 迭代记录和决策理由见 Notion 项目页。

一个开箱即用的长篇小说 AI 创作工具。单个可执行文件，内置完整 Web 界面，连接任意 OpenAI 兼容 API 即可自动生成大纲并逐章创作长篇小说。

## 与原版的主要差异

### 记忆系统

| 维度 | 原版 | 本版 |
|------|------|------|
| 存储 | progress.json 内嵌，无向量检索，无实体关联 | **双轨记忆架构**：轨道一（热记忆）保留原生 progress.json 逐章片段；轨道二（冷记忆）Mem0 Sidecar + BAAI/bge-small-zh-v1.5 本地向量检索 |
| 检索 | 无 | SQLite + FTS 全文 + BM25 排序 + 语义向量检索 + 实体关系图谱，检索时融合排序 |
| 权重 | 无 | 每 20 章自动重算实体权重（"选角"机制，近期重要角色优先） |
| 容错 | 无需考虑 | Mem0 不可用时优雅降级，走原生记忆兜底 |
| Agent 工具 | 无 | 新增 `search_memories` 会话式检索工具 |

### 写作人格

原版系统提示词为扁平规则：「你是一位小说作者。只输出正文。保持视角统一。」

本版改写为**三层人格架构**：

- **顶层**：身份三观（满脑子骚操作的年轻人、较真不教条）
- **中层**：表达风格偏好（画面 > 解释、玩梗不文学腔、绝不注水）
- **底层**：硬规则（不输元信息、视角统一）

`persona.txt` 文件覆盖可自定义。人格还影响记忆的过滤与诠释（选角联动：检索结果在人格框架下被筛选）。

### Prompt 分层注入

原版 user prompt 全部平铺：StorySynopsis / HistorySummary / PreviousEnding / WritingStyle 等无差别混排。

本版改为**三权分层注入**：

- 🔴 **核心指令**：本章大纲 · 叙述视角 · 字数
- 🟡 **约束**：前情 · 项目指导 · 伏笔
- 🟢 **参考**：完整大纲 · 角色 · 世界观 · 记忆

删除了 WritingStyle（已由系统人格覆盖）。PreviousEnding 移出写作 prompt，仅保留一致性检查。

### 验收流程

原版单维度事实核查 + 循环重试（最多 3 次 + 冲突分析 + 约束再试）。

本版改为**三维一次验收**：

| 维度 | 内容 |
|------|------|
| 事实核查 | 原版 6 条规则（人设/地点/时间线/因果/能力/逻辑） |
| Gate 7 情绪冗余 | 7 条检测规则：同一情绪多次声明/抽象情绪词包裹动作/场景后追加情绪总结/动作已表达情绪又写心理/套话比喻/表态后追加解释/跨段重复同一情绪。豁免：排比/刻意重复；因新事件再次激发同类情绪 |
| Gate 8 修辞自觉 | 5 条检测规则：修辞与场景格调一致/修辞浮夸或平淡与场景强度不符/同段修辞密度过高/角色专属修辞与身份一致/修辞服务于叙事目的 |

验收结果产出 GateReports[] 存入章节数据，前端展示 PASS / FAIL + 问题列表。
修订：一次 API 调用检测 3 个维度 → 有问题则汇总为一份反馈 → 一次性修订 → 不重新验证。不再循环重试。

### 生成流程调整

原版：生成正文 → 摘要 → 事实核查（循环）→ 伏笔更新 → 记忆同步 → 标 review

本版：生成正文 → 三维验收 → 可选修订 → **标 review（等待用户确认）** → 用户确认 → 异步：摘要生成 → 伏笔更新 → 记忆同步 → 标 accepted

摘要生成、伏笔状态更新、记忆同步从生成阶段移到确认阶段，节省用户在审阅前不必要的 token 消耗。

### 大纲处理

原版 StorySynopsis 标签为"故事梗概"，作为逐章写作指令直接注入 prompt。

本版改为两套分层机制：

- **按章节 AI 生成大纲**：不变，仍为逐章写作的核心指令
- **用户上传的完整大纲**：通过 `/api/outline/parse` 接口提取到角色/世界观/组织/关系/伏笔五个结构化接口。生成时判断接口是否有数据——有则不注入完整大纲文本，无则全量兜底。两者分层共存

### 其他改造

| # | 原版 | 本版 |
|---|------|------|
| 1 | 摘要字数无硬限制 | 代码层 500 字硬截断 |
| 2 | 无章节大纲关键词检索 | 新增 `.declarations/declarations.json` 纯关键词索引 + Agent 工具 `search_declarations`，按需调用不自动注入 |
| 3 | 无故事弧位置感知 | 新增 `get_narrative_position` Agent 工具 + 系统提示注入当前阶段 |
| 4 | craft 类 skill 与 polish 无差别自动注入 | craft 类过滤，仅 polish 类自动注入 |
| 5 | 后端缺 ensureProject 保护 / doConfirm 竞态 / 大纲编辑限制 | 全部修复 |
| 6 | 确认流程同步（仅标 accepted） | 改为异步（后台跑摘要+伏笔+记忆，完成后标 accepted） |
| 7 | story-deslop 为 6-Gate 检测 | 升级为 7-Gate（新增情绪冗余检测），prompts.go FactCheck 扩展为三维验收 |

## 功能亮点

- **单文件运行**：一个二进制文件 + 一个浏览器，无需安装数据库或其他依赖
- **多项目管理**：每部小说一个独立项目，可随时切换、新建、删除
- **两阶段创作**：先生成全书大纲供你审核修订，确认后逐章写作
- **逐章审核**：每章生成后展示三维验收报告（事实核查/情绪冗余/修辞自觉），可确认或修订
- **自动确认模式**：开启后 AI 自动逐章连写，无需手动逐章确认，可随时开关
- **结构化设定**：角色、世界观、组织、人物关系独立管理
- **关系图谱**：可视化展示角色/组织/世界观之间的关系网络
- **伏笔系统**：AI 规划伏笔方案，写作时注入活跃伏笔，完成后自动追踪超期告警
- **双轨叙事记忆**：每章确认后自动提取关键叙事细节，Mem0+原生双轨跨章节注入
- **三维验收**：事实核查 + 情绪冗余检测 + 修辞自觉，生成后一次 API 完成
- **大纲分叉架构**：完整大纲解析到接口，有数据时不注入 prompt
- **续写已有作品**：粘贴已有文本，AI 分析提取设定与章节摘要后接着往下写
- **全书优化**：完稿后一键诊断 → 一致性核查 → 生成优化工单 → 逐章自动修订
- **技能系统**：7-Gate 故事去味检测、写作技巧等内置技能，支持自定义
- **AI 助理**：内置聊天助理，可通过对话查询和修改设定/大纲/章节，破坏性操作有多层防护
- **三层人格系统**：`persona.txt` 覆盖，人格决定 AI 如何理解和运用注入的记忆
- **实时流式输出**：生成过程逐字实时展示
- **断点续作**：进度随时落盘，关闭后重新打开自动恢复
- **导出**：一键导出全书 TXT，章节文件以 Markdown 保存
- **多语言**：每个项目可选择中文或英文

## 快速开始

### 1. 获取程序

下载对应平台的发布版本，或自行编译（见文末「开发」一节）。

### 2. 运行

```bash
./show-me-the-story            # 数据保存在当前目录
./show-me-the-story ~/novels   # 或指定一个数据目录
```

启动后访问 `http://localhost:48090`（端口可通过环境变量 `PORT` 修改）。

### 3. 配置 API

首次进入会提示创建项目。创建后在「配置」页填写 API 地址、模型名称、API Key。API 配置全局共享，所有项目通用。

### 4. 开始创作

1. **配置故事**：填写故事类型、章节数、每章字数等；可让 AI 一键生成初始设定
2. **生成大纲**：AI 输出全书章节大纲，可整体提修订意见或逐章内联编辑
3. **逐章写作**：生成正文 → 三维验收报告（事实核查 + 情绪冗余 + 修辞自觉）→ 审核确认或修订
4. **想省事**：打开「自动确认」开关，AI 自动逐章连写到底
